\section*{Independent Review --- ACM TOMM Submission Readiness (Round 5)}

\subsection*{Overall Assessment}
The manuscript is at submission quality for ACM TOMM. The contribution is clearly positioned as a calibrated privacy--utility trade-off optimizer rather than a universal privacy guarantee, the experimental protocol is reproducible and self-consistent, and the limitations are explicitly stated. Four prior independent review rounds found only editorial-level issues, all of which have been resolved.

This round identifies one critical scientific-consistency issue (now fixed), three minor presentation items (now fixed), and confirms that no further structural, methodological, or experimental changes are needed.

\textbf{Recommendation:} Accept with no further revisions required.

\subsection*{Issues Found and Resolved in This Round}

\subsubsection*{R1 (Critical). Incorrect ``parameter efficiency'' claim in matched-operating-point analysis}
The matched-operating-point section (Section~4.5) previously stated that PPEDCRF ``reaches the same utility target with a smaller nominal sigma.'' Table~7 shows the opposite: PPEDCRF requires \emph{larger} $\sigma_0$ than global Gaussian noise to reach the same PSNR (e.g., $\sigma_0=8$ vs.\ 4 at $\approx$36\,dB). The correct interpretation is that at any given $\sigma_0$, PPEDCRF preserves approximately 6\,dB more PSNR by concentrating noise on the DCRF support.

\textbf{Status:} Fixed. The matched-operating-point paragraph and the abstract have been rewritten to state the correct directional claim.

\subsubsection*{R2 (Minor). Global Gaussian frontier PSNR values not in any table}
Section~4.4 quotes specific Global Gaussian PSNR values at $\sigma_0=16$ (24.23\,dB) and $\sigma_0=24$ (20.83\,dB) that appear only in Figure~3 and the released CSV, not in any numbered table.

\textbf{Status:} Fixed. A parenthetical note now states that these values come from the frontier data plotted in Figure~3 and available in the released benchmark outputs.

\subsubsection*{R3 (Minor). Appendix terminology inconsistency}
Appendix Section~2 used ``fixed ResNet18 attacker'' in the benchmark construction steps, whereas the main paper consistently uses ``backbone'' or ``embedder'' in operational contexts.

\textbf{Status:} Fixed. Changed to ``fixed ResNet18 backbone'' in the appendix.

\subsubsection*{R4 (Minor). Figure~3 caption not self-contained}
The caption said ``a substantially better operating curve'' without quantifying the advantage.

\textbf{Status:} Fixed. Caption now states ``approximately 6\,dB PSNR advantage.''

\subsubsection*{R5 (Carried from prior round). ``Resolves instability'' language}
Two instances in the robustness analysis used ``resolves this instability'' / ``is resolved'' regarding CLIP ViT-L/14 seed averaging, which is too strong given MixVPR remains an exception.

\textbf{Status:} Fixed in this cycle. Changed to ``substantially reduces'' in both locations.

\subsection*{Confirmed Strengths (No Changes Needed)}
\begin{itemize}
    \item Contribution positioning: calibrated trade-off optimizer, not universal privacy.
    \item Unified 8-backbone, 3-seed benchmark protocol with single output directory.
    \item Matched-operating-point analysis with three PSNR targets.
    \item Explicit MixVPR adverse-transfer exception documented throughout.
    \item 50-pair appendix scaling confirmation.
    \item Temporal consistency metrics with honest interpretation.
    \item All table captions specify deterministic vs.\ stochastic reporting conventions.
    \item DP-style calibration clearly distinguished from formal DP guarantee.
    \item Runtime performance noted ($<$50\,ms per frame).
\end{itemize}

\subsection*{Remaining Known Limitations (Not Actionable Without New Data)}
\begin{itemize}
    \item Proxy benchmark based on visual similarity rather than GPS ground truth.
    \item No large-scale cross-view or street-view evaluation.
    \item MixVPR adverse transfer not resolved.
    \item 4 minor BibTeX warnings (missing page numbers for 3 preprint-style entries).
\end{itemize}

These are explicitly acknowledged in the paper and do not block submission.
