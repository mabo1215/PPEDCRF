\subsection*{Major Concerns}

\subsubsection*{M1. Protocol-consistency gap between text claims and robustness source data}
The manuscript states protocol-level averaging over multiple seeds in the experimental section, while the newly unified 8-backbone robustness table appears to come from a single-seed run (all standard deviations are effectively absent and the output summary indicates one seed).

\textbf{Why this matters:} reviewers will treat this as an evidence-integrity issue, not just a formatting issue.

\textbf{Required revision:}
\begin{itemize}
  \item Harmonize protocol text with actual robustness data generation.
  \item Either (A) rerun the unified 8-backbone table with the stated multi-seed setting, or (B) explicitly label the unified robustness table as single-seed and scope conclusions accordingly.
  \item Ensure all sections that cite robustness claims use the same evidence scope.
\end{itemize}

\subsubsection*{M2. Mixed benchmark tracks still need cleaner separation in interpretation}
The manuscript combines (i) default ablation/frontier values from one benchmark configuration and (ii) unified 8-backbone robustness from a larger paired-scene candidate setting. This is acceptable if transparent, but current writing can still be misread as a single homogeneous experiment block.

\textbf{Required revision:}
\begin{itemize}
  \item Add one explicit paragraph clarifying that ablation/frontier/matched and unified robustness come from different benchmark settings.
  \item State what remains directly comparable and what should be interpreted only directionally.
\end{itemize}

\subsubsection*{M3. Heterogeneous transfer interpretation improved but still needs quantitative guardrails}
The draft now acknowledges adverse transfer rows (e.g., CLIP ViT-L/14), but still uses broad positive framing in nearby summary text.

\textbf{Required revision:}
\begin{itemize}
  \item Add a short quantitative summary sentence after Table~4 with counts of negative/neutral/positive $\Delta$ cases.
  \item Keep the final takeaway explicitly bounded: support transfer is useful but not universally protective.
\end{itemize}

\subsubsection*{M4. Larger paired-scene requirement is only partially satisfied}
The manuscript now uses a larger candidate pool in unified runs, but the number of query pairs in the robustness table remains small. The current claims are still proxy-bounded.

\textbf{Required revision:}
\begin{itemize}
  \item If compute permits, add one larger-pair robustness rerun and report only the key summary deltas.
  \item If compute does not permit in this cycle, state that larger-pair confirmation is pending and reduce wording strength accordingly.
\end{itemize}
